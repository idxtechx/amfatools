# Contributing to Amfatools 🚀

First off, thank you for considering contributing to Amfatools! It's people like you who make this SDK better for the Solana community.

## How Can I Contribute?

### Reporting Bugs
- Open an **Issue** on GitHub.
- Describe the bug and include steps to reproduce it.
- Mention your Node.js version and any error logs.

### Suggesting Enhancements
- Open an **Issue** with the tag `enhancement`.
- Explain why this feature would be useful for the SDK or the Bots.

### Pull Requests (Code Contributions)
1. **Fork** the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`.
3. Make your changes and ensure your code follows the existing style.
4. **Commit** your changes: `git commit -m 'feat: add new monitoring logic'`.
5. **Push** to the branch: `git push origin feature/your-feature-name`.
6. Open a **Pull Request**.

## Project Structure
- `bot-*.js`: Standalone Telegram Bots.
- `index.js`: Main SDK Entry Point.
- `sdk/`: Core logic modules.

## Style Guide
- Use clear and descriptive variable names.
- Ensure no API Keys or sensitive data are committed.
- Add comments for complex Solana logic.

Thank you for your support!
